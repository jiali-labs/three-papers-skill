5. Compliance changes made to SKILL.md
--------------------------------------

Added a new "合规与使用边界" section near the top of SKILL.md.

Key excerpt:

```markdown
## 合规与使用边界

本 Skill 仅用于学习资料推荐和入门认知框架搭建，不构成完整文献综述，也不构成投资、医疗、法律、升学、职业选择等专业建议。

不得提供、复制、存储、转载论文全文或大段原文；不得帮助用户绕过付费墙、登录限制、出版社限制或非公开访问限制。只能推荐论文、提供公开页面线索，并基于公开可验证信息做简要介绍。

涉及金融、医学、法律、健康、投资等高风险领域时，只能推荐学习材料和阅读路径，不得给出交易建议、诊断建议、治疗建议、法律判断或具体决策建议。

如果用户输入涉及违法违规、侵犯隐私、绕过平台规则、危险行为、仇恨暴力、成人内容、医疗诊断、投资交易建议等不适合处理的主题，应拒绝推荐论文，并提示用户换一个合法、安全、适合学习资料推荐的领域。
```

Workflow now starts with compliance/safety handling before mode selection:

```markdown
2. 先做合规与安全判断；遇到不适合处理的主题，拒绝推荐论文并给出安全改写方向。
```

6. Evidence/source wording changes
----------------------------------

Changed evidence/source wording to avoid implying copyrighted full-text retrieval or dependency on fixed third-party APIs.

Before:

```markdown
优先使用 arXiv、论文 PDF、官方会议/期刊页面、出版社页面、作者主页、Semantic Scholar、Google Scholar、DBLP、ACL Anthology、OpenReview 等可靠来源。

- Level 1：可访问论文全文、PDF、arXiv 页面、官方会议页或出版社页。可以较高置信讨论论文。
```

After:

```markdown
不要把本 Skill 表述成依赖某个固定外部 API 或第三方服务；如果当前运行环境支持访问公开网页，可优先参考公开论文页面、官方会议页、出版社摘要页、作者公开页面、arXiv / OpenReview 等公开页面和可靠元信息；如果当前环境无法访问或核验来源，必须说明没有进行实时核验，并给出保守输出。

- Level 1：可核验公开论文页面、官方会议页、出版社页面、arXiv / OpenReview 等公开页面，能确认标题、作者、年份、来源和摘要/简介等信息。可以较高置信地做入门级介绍，但不得提供论文全文或大段原文。
```

Preserved internal-only Level rule:

```markdown
证据分级 Level 1/2/3/4 只用于内部判断，不要在用户可见输出中直接出现「Level 1」「Level 2」「Level 3」「Level 4」等字样。
```

Updated output template:

```markdown
信息依据：
{用自然中文说明公开来源、核验范围和访问限制，不显示内部 Level 标签}
```

7. Professional advice / high-risk field boundaries added
---------------------------------------------------------

SKILL.md now explicitly says the Skill does not provide professional decisions for:

- investment
- trading
- medical
- legal
- health
- education/admission
- career choice

For finance, medicine, law, health, and investment, the Skill may only recommend learning materials and reading paths.

8. Unsafe-topic refusal rules added
-----------------------------------

SKILL.md now instructs refusal for topics involving:

- illegal activity
- privacy invasion
- bypassing platform rules
- dangerous behavior
- hate/violence
- adult content
- medical diagnosis
- investment/trading advice

The fallback should ask the user to choose a legal, safe, learning-oriented field.
