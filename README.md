# three-papers-skill

Skill 名称：三篇论文入门

一句话介绍：输入一个新领域，推荐 3 篇帮助小白建立认知框架的经典论文。

## 目标用户

这个 Skill 面向中文用户，尤其是小红书上想快速理解一个新领域、但还不知道从哪里开始读论文的新手。

默认用户画像：没有专业背景，但愿意认真学习；希望先搭建「第一版认知框架」，而不是立刻做完整文献综述。

## 这个 Skill 做什么

- 根据用户自由输入的领域或方向，判断是否适合用 3 篇论文入门。
- 对过宽领域先收窄学习主线，再推荐论文。
- 对新兴方向推荐代表性论文，但不夸大为「公认经典」。
- 对偏实践、商业、运营或个人成长的主题，拒绝强行推荐学术论文，并给出更合适的改写方向。
- 用中文解释每篇论文解决的问题、推荐理由、小白阅读方式和读完应建立的认知。
- 强制披露论文信息依据，避免编造论文、链接、作者、年份或实验细节。

## 这个 Skill 不做什么

- 不做完整系统综述。
- 不保证推荐是唯一正确答案。
- 不按引用量或发布时间机械排序。
- 不为所有输入强行凑 3 篇论文。
- 不在无法核验信息时编造论文元数据或链接。
- 不把商业运营类问题伪装成学术论文入门问题。

## 示例输入

- 大语言模型
- AI Agent
- 推荐系统
- 扩散模型
- 量化交易
- 新能源汽车
- 小红书运营
- 我想了解脑机接口，应该先读哪三篇论文？

## 输出结构

适合论文入门的主题会输出：

1. 你想了解的领域
2. 领域判断
3. 推荐逻辑
4. 阅读顺序
5. 三篇论文共同建立的认知框架
6. 每篇论文的标题、作者、年份/来源、链接、问题解释、推荐理由、阅读建议、认知收获、信息依据
7. 可选的「为什么没有选其他常见论文」
8. 下一步怎么学

不适合论文优先的主题会输出：

1. 为什么不适合直接推荐 3 篇经典论文
2. 可以怎样改写成更学术或技术化的问题
3. 如果目标是实践入门，更适合看的资料类型

## 领域判断模式

- Mode A：高置信研究领域，例如大语言模型、推荐系统、扩散模型、强化学习、脑机接口。
- Mode B：过宽或多分支领域，例如人工智能、金融、芯片、新能源、新能源汽车、心理学。
- Mode C：快速发展且经典尚未完全沉淀的方向，例如 AI Agent、具身智能、World Model、AI for Science。
- Mode D：不适合论文优先的实践型主题，例如小红书运营、职场沟通、商业模式、个人成长、品牌营销。

## 证据与反幻觉政策

Skill 要按以下证据层级处理论文信息：

- Level 1：可访问论文全文、PDF、arXiv 页面、官方会议页或出版社页。
- Level 2：可访问摘要和可靠元信息。
- Level 3：只能确认标题、作者、年份、来源等基础信息。
- Level 4：无法可靠确认。不得进入最终 3 篇推荐。

每篇推荐论文必须包含「信息依据」。如果无法联网或来源访问受限，必须说明限制；如果信息不足，宁可不给最终名单，也不能编造。

## MVP 限制

- 当前版本只提供 Skill 指令和人工测试用例，不包含自动化检索脚本。
- 论文核验依赖使用者当时可用的网络、数据库或上下文资料。
- 对跨学科领域的收窄主线可能需要根据用户背景继续调整。
- 中文解释偏入门，不替代专业课程、教材或系统综述。

## 未来改进

- 增加常见领域的参考候选池，但仍保留联网核验要求。
- 增加自动化检查脚本，验证输出是否包含信息依据、链接披露和模式判断。
- 为小红书发布场景增加更短的卡片版输出。
- 增加「按用户背景改写」模板，例如产品经理、投资人、工程师、研究生。
